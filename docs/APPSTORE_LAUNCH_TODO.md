## Legal placeholders (BLOCKING App Store + Play Store submission)

`apps/web/src/screens/LegalScreen.tsx` still contains two unfilled
template tokens. The static HTML at `apps/web/public/{privacy,terms}.html`
inherits them, so reviewers will see the literal `{{COMPANY_LEGAL_NAME}}`
and `{{JURISDICTION}}` strings. Fill them before tagging `v1.0.0-rc1`.

| Token | Where it shows | What to fill |
| ----- | -------------- | ------------ |
| `{{COMPANY_LEGAL_NAME}}` | Privacy "Who we are"; Terms "Welcome", "Our content" | The legal-entity name on file with Apple Developer + Google Play (e.g. "Aqil Lakhani Sole Proprietorship" or your registered LLC) |
| `{{JURISDICTION}}` | Terms "Disputes — binding arbitration" | The state/province whose courts handle disputes (typically your home state for a sole prop, or where the LLC is registered) |

`{{CONTACT_EMAIL}}` was already filled with `support@gethaloframe.com`.

### One-shot replacement

After you decide on values, run from repo root:

```bash
sed -i 's/{{COMPANY_LEGAL_NAME}}/Aqil Lakhani/g' apps/web/src/screens/LegalScreen.tsx
sed -i 's/{{JURISDICTION}}/Ontario, Canada/g' apps/web/src/screens/LegalScreen.tsx
node scripts/build-legal.mjs
git add apps/web/src/screens/LegalScreen.tsx apps/web/public/privacy.html apps/web/public/terms.html
git commit -m "fix(legal): fill {{COMPANY_LEGAL_NAME}} + {{JURISDICTION}}"
```

The unit tests for `build-legal` will not catch a missed placeholder
(by design — these are content-only fills). To manually verify:

```bash
grep -c '{{' apps/web/public/privacy.html apps/web/public/terms.html
# Expected: both files report 0
```
